History of <html> and {css}

Evolution of <html>

HTML (HyperText Markup Language) was created in late 1990 by Tim Berners-Lee. At the time he was working at CERN and propose his markup language to CERN as a way for scientists to share files. Berners-Lee and data system engineer Robert Cailliau, requested funding from CERN, but it was not granted. The first document that included HTML markup was called "HTML Tags" and is available here. The tags in HTML Tags are fairly simple compared to the tags available in today’s HTML 5. There have been many versions of HTML, with each one bringing new features and new methods of designing websites.

The first version of HTML was released in 1993. This version of HTML had very basic features only having the capability of displaying text and images. The very few tags that it did have were the <P>, the <H1> to <H2>, <UL> and <A> tags. It had no facility for styling such as changing text fonts or colours.

In 1995, HTML 2 was released. This version of HTML was a standardised version of HTML that all future versions of HTML would build upon. It had all the capabilities of the first version but with the addition some new HTML tags. Some of the new tags that were introduced were the <HEAD>, <BODY>, <TABLE> and the <LINK> tag which gave the ability to link to other documents. During this time the W3C (World Wide Web Consortium) was formed to ensure that a uniform standard was established and maintained among the web browsers.

HTML 3.2 was developed in 1997. As with all versions of HTML there were improvements on various aspects of the language. Support CSS embedded in HTML was added which allowed web developers to have greater freedom when it came to styling a website. Some notable HTML tags that were introduced were the <DIV> <TITLE>, <SCRIPT> and <STYLE> tags which were reserved for use in the future with scripting languages and stylesheets respectively.

HTML 4.01 was released in 1999 and with it came support of external style sheets. This was a very important development in terms of web-design. Support for style sheets meant that CSS could be applied to the entire website rather than for each individual HTML file. Some of the tags that were introduced in 4.01 were the <BUTTON>, <IFRAME> and the <SPAN> tags.

HTML 5 was initially released in 2008 but in 2014 there was a major update and a W3C recommendation. With this new iteration came a few changes. All the previous versions of HTML used tags in uppercase, but now with HTML 5 lowercase for tags can be used. HTML 5 added various additional tags such as <article>, <header>, <footer> and quite a few others. Since 2014 there have been two new HTML 5 versions, HTML 5.1 and 5.2.

Evolution of {css}

Like with HTML, there have been many different iterations of CSS through the past few years. CSS (Cascading Style Sheets) is used to style a HTML webpage. There are three different CSS standards, from CSS to CSS3. Håkon Wium Lie first proposed CSS on the 10th of October, 1994 and the first draft of CSS is still available here. During this time, he was working at CERN with Berners-Lee. Bert Bos joined Wium Lie in developing CSS. Bos at that time been creating a customisable browser with style sheets called Argo. Many features from Argo eventually made their way into CSS.

In 1996, CSS1 became an official W3C recommendation. This version of CSS had the ability to change the typeface, the colour of text, backgrounds, and other elements such as the spacing between the words and letters, how text, images and tables are aligned and introduced the ability to change the borders, margins and positioning of elements as well as many other capabilities.

CSS2 was developed as a W3C recommendation in May of 1998. This version of CSS featured new capabilities such as absolute, fixed, relative positioning and also the z-index. CSS2 also allowed various media types and different font options such as shadow.

It took a considerable amount of time for CSS2.1 to be published as a W3C after going to and from working draft status to candidate recommendation status. CSS2.1 fixed errors that were present in CSS2 and also removed some redundant features. CSS2.1 was finally published as a W3C recommendation on the 7th of June 2011.

For CSS3 the W3C introduced modules, stating that it was simpler to update individual parts of CSS rather than to add many updates into one single specification. There are now many modules in the CSS specification such as: css3-backround, css3-color, css-fonts-3 and css3-box, among others.
